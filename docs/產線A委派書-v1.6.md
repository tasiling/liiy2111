## 〇、總則

1. 欄位表為**封閉清單**:不得新增、刪除、更名任何欄位。認為有缺漏時,停下回報,不得自行補充。
2. 選項型欄位的選項值為**封閉清單**:必須與本文件完全一致,不得增減或改寫措辭。
3. 建置完成後,每個資料庫填入本文件指定的測試資料,供驗收。
4. 所有資料結構以「未來可被網站程式調用」為標準:識別碼唯一、關聯明確、避免自由文字承載結構化資訊。
5. Notion 不支援的機制(select 預設值、數字範圍限制、條件式預設)**不視為缺陷、不需變通實作**:已定案由產線 C 的 App 於寫入時強制,過渡期靠 SOP 與手動。執行時建成一般欄位即可。
6. **本產線已完成建置**。v1.4 起之異動皆為既有資料庫上的增量修改,非重建。
7. **能以資料列承載者,不擴充欄位**(v1.5 原則)。語氣指引即為實例:它是跨規則共用的單一內容,以 DB-14 一筆資料存放,**不在 DB-05 新增欄位**。

## 二、關聯總圖(文字版)

```
DB-01 牌組 ←─ DB-02 牌卡 ←─ DB-04 抽牌明細 ─→ DB-03 Session ─→ DB-05 規則庫(版本自關聯)
                                                  ↑        ↑
DB-13 實驗庫 ──投放紀錄──────────────────────────┘        │
DB-08 月主題包 ──來源──────────────────────────→ DB-03    │
DB-09 回饋 ──對象──→ DB-03 / DB-05 ←──目標── DB-10 提案──┘
DB-06 固定項目 ←─ DB-07 序列節點 ─→ DB-05 / DB-12 素材庫 ─→ DB-06
DB-16 標籤詞庫 ←─關聯── DB-08 月主題包/DB-11 原子庫/DB-12 素材庫(日記箱)/DB-14 知識庫

DB-14 知識庫 ──對應服務──→ DB-11 服務原子庫
```

標籤匹配機制一律透過 DB-16 的關聯比對,不再使用多選字串比對。新增詞彙只在 DB-16 一處進行。服務歸屬則透過 DB-14 → DB-11 的直接關聯,不經標籤。

## 三、測試資料(驗收用,每庫至少建立)
