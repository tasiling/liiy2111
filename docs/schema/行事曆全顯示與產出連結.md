# P1 行事曆同日全顯示 + 跳轉、P5 產出連結(擁有者 2026-07-12 追加指示)

## P1 行事曆:同日多筆任務全數顯示

原本月曆網格每天只預覽前 2 筆,超過以「+N 更多」摺疊,需點格子才能在下方看完整清單。
擁有者指出實測情境(7/13–26 每日 2–3 筆)下這樣會讓格內看不到全部任務,改為：**格內堆疊
顯示當天全部任務**,不再截斷(`app/page.tsx` 移除 `MAX_PREVIEW_PER_DAY` 裁切邏輯)。
今日待辦區(`todayTasks`)本來就是列出當天全部到期明細,未截斷,不需改動。

## P1 行事曆任務點擊跳轉 P5

行事曆任務標籤(月曆格內、今日待辦、選取日展開清單三處共用 `TaskRow`/格內項目渲染)新增
點擊跳轉:type=「明細」的項目若有所屬 Session,渲染為 `<Link href="/sessions?sessionId=..." >`,
點擊導向 P5 頁面並自動展開、捲動定位到該 Session。type=「場次」的項目不屬於任何 Session,
維持純文字不可點擊。

`GET /api/dashboard` 的 `calendarFromDetails` 新增回傳 `所屬Session`(明細所屬 Session 的
page id)供前端組跳轉連結,`場次` 類型項目本無此欄位。

P5(`app/sessions/page.tsx`)讀取網址參數 `sessionId`:

- 若目標在近期 60 天清單內:自動展開明細、捲動定位、外框高亮。
- 若不在清單內(如建立日期超過 60 天前):改打新增的 `GET /api/sessions/[id]` 補查表頭,
  插入清單最前面顯示,同樣展開/定位/高亮。
- 查無此筆(ID 錯誤或已刪除):顯示提示訊息,不阻斷頁面其餘功能。

`useSearchParams()` 需要 Suspense 邊界(Next.js App Router 規定,CSR bailout 情境),
`SessionsPage` 拆成外層 `Suspense` 包裹 + 內層 `SessionsPageInner` 實際邏輯。

## P5 表頭「產出連結」檢視與編輯

DB-03 本來就有「產出連結」欄(type=url,description=「內容成品位置」),先前程式碼未讀寫。
本次補上:

- `lib/notion/queries.ts` 的 `mapSession()` 新增回傳 `產出連結`。
- `lib/notion/mutations.ts` 新增 `updateSessionOutputLink()`,搭配新路由
  `PATCH /api/sessions/[id]/output-link`。純欄位覆寫,不牽動狀態機,可清空(用於撤銷誤填)。
  Notion url 屬性清空須寫 `null`,不能寫空字串(空字串會被判為不合法 URL 格式)。
- P5 每筆 Session 表頭下方新增 `OutputLinkEditor`:未編輯時顯示現有連結(可點開)或「未登記」,
  按「編輯」切換成輸入框,支援手動補登記內容完成後的連結網址(如發布後貼上連結)。
