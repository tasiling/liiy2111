# 介面用語抽換層(擁有者 2026-07-14 追加指示)

## 規則

所有頁面/模組的顯示名稱一律經 `lib/labels.ts` 的 `pageLabel()` 輸出,不寫死在元件裡。
資料庫欄位名與選項值(如 DB-03「項目用途」「狀態」的選項字串、DB-05「適用項目」等)
完全不受影響——這是純顯示層,只換畫面上看得到的文字,不動 Notion schema、不動任何
內部程式碼識別碼。

初版對照(委派書 P 編號 → 介面顯示文字):

| 內部代號 | 對應頁面 | 介面顯示文字 |
|---|---|---|
| P1 | `/`(主控台首頁) | 總覽台 |
| P2 | `/expand`(序列展開引擎) | 序列展開 |
| P3 | `/production-day`(生產日工作台) | 生產日工作台 |
| P4 | (尚未實作,第三期範圍) | 活動註冊表 |
| P5 | `/sessions`(Session 工作站) | 任務管理站 |
| P6 | `/feedback`(回饋快填) | 快速評分 |
| P7 | (尚未實作,第三期範圍) | 提案審核區 |
| P8 | `/generate`(P8 組稿) | 組稿台 |

擁有者稍後會分批提供品牌用語替換清單,屆時只需要改 `PAGE_LABELS` 這個物件,不用動
任何頁面元件——這正是設這層的目的。

## 套用範圍(本批實際改動)

- `app/layout.tsx` 導覽列的六個連結文字,全部改成 `pageLabel(...)`。
- 各頁面自己的 `<h1>` 標題(P1 首頁原本沒有頁面標題,本次補上;P8 組稿頁保留階段
  說明後綴「— 階段一・一鍵組稿」,因為那是實作階段描述,不是頁面代號名稱,不屬於
  這次對照表的範圍)。
- `app/production-day/page.tsx` 的 SOP 步驟清單中,凡是指向其他頁面的連結標籤
  (如「序列展開」「快速評分」「組稿台」)與步驟描述文字內提到其他模組名稱處,一律
  改用 `pageLabel(...)` 組字串,不再寫死中文。

## 本批未涵蓋、刻意不動的範圍

- **內容規則/方法名稱/服務名稱**(如 P8-0 表單的「維度」「方法」選單、DB-14 服務
  導流占卜配對表顯示文字):擁有者明確指示這批正在實跑驗證中,14 天後統一升版,
  現在動會返工,本次完全沒有觸碰這些檔案的顯示文字。
- **每個元件內部的一般性文案**(按鈕文字、錯誤訊息、表單提示語等):這次只處理
  「頁面/模組名稱」這一批有給明確初版對照的範圍。若日後品牌用語清單也涵蓋這些
  一般性文案,需要另外評估是否比照這個模式抽出到對照檔,或該用其他方式(如 i18n
  慣用的 key-value 訊息檔)處理——規模與這次的 8 個頁面代號完全不同量級,屬於
  未涵蓋情況,屆時再另行報告討論範圍,不在本次自行擴大處理。
